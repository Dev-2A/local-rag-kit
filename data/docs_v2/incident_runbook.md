# 장애 대응 런북 (Incident Runbook)

## 우선순위
1) 영향 범위(사용자/서비스/매출) 파악
2) 최근 변경(배포/설정/배치) 확인
3) 로그/메트릭/트레이스 확인

## 즉시 조치
- 장애 공지: 커뮤니케이션 채널 고정(슬랙/전화), 타임라인 기록
- 임시 완화: 트래픽 차단, 기능 플래그 off, 캐시 우회/적용
- 롤백: 배포 파이프라인에서 이전 태그로 되돌린다

## 체크리스트
- 에러율(5xx), 지연시간, 트래픽 급변 확인
- 의존성 장애(DB/Redis/외부 API) 여부 확인
- 재현 가능 여부 및 재현 조건 정리
